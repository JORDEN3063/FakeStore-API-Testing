# FakeStore API Testing

## Project Overview
This project demonstrates API testing using Postman on the FakeStore API, which simulates an e-commerce backend system.

The goal of this project is to validate API endpoints and verify that the application returns correct responses for different requests.

## Tools Used
- Postman
- REST API
- JSON

## API Operations Tested
- GET – Retrieve product details
- POST – Create a new product
- PUT – Update product information
- DELETE – Remove a product

## Test Scenarios
- Retrieve all products
- Retrieve a specific product by ID
- Create a new product
- Update product information
- Delete a product

## Validations Performed
- Status code validation
- Response body validation
- JSON structure verification
- Positive and negative test scenarios

## Example Status Codes Verified
- 200 – Success
- 201 – Resource created
- 404 – Resource not found
- 400 – Bad request

## How to Use
1. Import the Postman collection into Postman
2. Execute the API requests
3. Verify responses and status codes
